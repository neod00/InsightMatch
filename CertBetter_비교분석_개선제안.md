## 🔍 CertBetter 주요 기능 분석

### 1. 컨설턴트 디렉토리 기능

#### 1.1 고급 필터링 시스템
- **Verified providers only**: 검증된 제공자만 필터링
- **Rated providers only**: 평가가 있는 제공자만 필터링
- **Standard (ISO 규격)**: ISO 9001, 14001, 45001 등 다중 선택
- **Service Type**: 서비스 유형별 필터링
- **Industry**: 산업별 필터링
- **Location**: 지역별 검색 (예: "Melbourne, Australia")

#### 1.2 검증 시스템
- **독립적 검증 배지**: "independently verified by CertBetter for operational legitimacy and transparency"
- 컨설턴트 카드에 검증 상태 명확히 표시
- 검증된 컨설턴트 우선 노출

#### 1.3 컨설턴트 등록 프로세스
- **"List Your ISO Business"** 버튼으로 등록 유도
- 등록 후 검증 프로세스 거쳐야 함
- 검증 완료 시 배지 부여

#### 1.4 컨설턴트 프로필 표시
- 회사명/이름
- 검증 배지
- "View Profile" 버튼으로 상세 정보 확인 가능
- 비교 기능 (Compare ISO Consultants)

## 💼 현재 InsightMatch 기능 현황

### 1. 컨설턴트 등록 기능
✅ **구현됨**
- `consultant_register.html`: 컨설턴트 등록 폼
- 기본 정보, ISO 규격, 산업 경험, 자격 및 경력 입력
- 등록 시 `verified=False` 기본값으로 설정

### 2. 데이터 모델
✅ **구현됨**
```python
class Consultant:
    verified = db.Column(db.Boolean, default=False)  # 검증 상태
    trust_score = db.Column(db.Float, default=0.0)   # 신뢰 점수
    iso_experience = db.Column(db.Text)              # ISO 경험
    industry_experience = db.Column(db.Text)         # 산업 경험
    # ... 기타 필드
```

### 3. 검색/필터링 API
✅ **부분 구현됨**
- `/api/consultants` 엔드포인트에서 필터링 지원
- `industry`, `iso`, `project_type`, `region` 파라미터 지원
- 하지만 **UI에서 필터링 인터페이스 부재**

### 4. 관리자 기능
⚠️ **부분 구현됨**
- `admin.html`: AI 분석 요청 관리만 가능
- **컨설턴트 승인/검증 기능 없음**

## 🔄 차이점 분석

### 1. 필터링 UI/UX

| 항목 | CertBetter | InsightMatch | 차이점 |
|------|-----------|--------------|--------|
| 필터 UI | 명확한 필터 패널 (Show Filter 버튼) | 없음 | **필터 UI 전혀 없음** |
| 검증 필터 | "Verified providers only" 체크박스 | API만 지원, UI 없음 | **UI 구현 필요** |
| 평가 필터 | "Rated providers only" 체크박스 | 없음 | **추가 필요** |
| ISO 규격 필터 | 드롭다운 다중 선택 | API만 지원 | **UI 구현 필요** |
| 산업 필터 | 드롭다운 | API만 지원 | **UI 구현 필요** |
| 지역 필터 | 텍스트 입력 (예: "Melbourne, Australia") | API만 지원 | **UI 구현 필요** |

### 2. 검증 시스템

| 항목 | CertBetter | InsightMatch | 차이점 |
|------|-----------|--------------|--------|
| 검증 배지 | "independently verified" 배지 명확히 표시 | `verified` 필드만 있음, UI 표시 없음 | **배지 UI 구현 필요** |
| 검증 프로세스 | 명확한 검증 절차 (운영 합법성, 투명성) | 관리자 승인 기능 없음 | **관리자 승인 기능 필요** |
| 검증 상태 표시 | 카드에 명확히 표시 | 없음 | **UI 표시 필요** |

### 3. 컨설턴트 등록

| 항목 | CertBetter | InsightMatch | 차이점 |
|------|-----------|--------------|--------|
| 등록 유도 | "List Your ISO Business" 버튼 | "컨설턴트 등록" 링크만 있음 | **더 눈에 띄는 CTA 필요** |
| 등록 후 프로세스 | 검증 대기 → 승인 → 배지 부여 | 등록 후 관리자 승인 기능 없음 | **승인 워크플로우 필요** |

### 4. 컨설턴트 표시

| 항목 | CertBetter | InsightMatch | 차이점 |
|------|-----------|--------------|--------|
| 프로필 상세 페이지 | "View Profile" 버튼으로 상세 정보 | 없음 | **프로필 상세 페이지 필요** |
| 비교 기능 | "Compare ISO Consultants" 기능 | 없음 | **비교 기능 추가 고려** |

## 🚀 개선 및 추가 기능 제안

### 🔴 우선순위 1: 필수 기능 (즉시 구현 필요)

#### 1.1 필터링 UI 구현
**현재 상태**: API는 있지만 UI가 없어 사용자가 필터링 불가능

**제안 사항**:
- 컨설턴트 리스트 상단에 필터 패널 추가
- 필터 옵션:
  - ✅ 검증된 컨설턴트만 보기 (체크박스)
  - ✅ 평가가 있는 컨설턴트만 보기 (체크박스)
  - ✅ ISO 규격 선택 (다중 선택)
  - ✅ 산업 선택 (다중 선택)
  - ✅ 지역 검색 (텍스트 입력)
  - ✅ 서비스 유형 선택 (신규/갱신/전환/통합)

**구현 위치**: `index.html`의 컨설턴트 리스트 섹션

#### 1.2 검증 배지 UI 구현
**현재 상태**: `verified` 필드가 있지만 UI에 표시되지 않음

**제안 사항**:
- 컨설턴트 카드에 검증 배지 표시
- 검증된 컨설턴트: "✓ 검증됨" 배지 (초록색)
- 미검증 컨설턴트: "검토 대기" 또는 배지 없음
- 검증 배지 클릭 시 검증 정보 팝업 표시

**구현 위치**: `script.js`의 `renderConsultants()` 함수

#### 1.3 관리자 컨설턴트 승인 기능
**현재 상태**: 관리자 페이지에 컨설턴트 승인 기능 없음

**제안 사항**:
- 관리자 페이지에 "컨설턴트 승인" 탭 추가
- 대기 중인 컨설턴트 목록 표시
- 각 컨설턴트의 등록 정보 확인
- 승인/거부 버튼
- 승인 시 `verified=True`로 변경 및 이메일 알림

**구현 위치**: `admin.html`에 새 섹션 추가

#### 1.4 컨설턴트 프로필 상세 페이지
**현재 상태**: 컨설턴트 상세 정보를 볼 수 있는 페이지 없음

**제안 사항**:
- `consultant_profile.html` 새로 생성
- 컨설턴트의 모든 정보 표시:
  - 기본 정보 (이름, 경력, 평점)
  - ISO 규격별 경험
  - 산업별 경험
  - 주요 프로젝트 이력
  - 자격증 정보
  - 검증 상태
- "문의하기" 버튼

**구현 위치**: 새 파일 생성

### 🟡 우선순위 2: 중요 기능 (단기 구현)

#### 2.1 평가/리뷰 시스템
**현재 상태**: `rating`, `reviews` 필드는 있지만 실제 리뷰 작성 기능 없음

**제안 사항**:
- 프로젝트 완료 후 리뷰 작성 기능
- 평점 및 후기 표시
- "Rated providers only" 필터 적용

#### 2.2 지역 기반 검색 강화
**현재 상태**: `regions` 필드가 있지만 활용도 낮음

**제안 사항**:
- 등록 시 지역 선택 (드롭다운 또는 지도)
- 지역별 필터링 강화
- "내 지역 컨설턴트" 우선 표시 옵션

#### 2.3 컨설턴트 비교 기능
**제안 사항**:
- 여러 컨설턴트 선택하여 비교
- 비교 테이블로 표시 (ISO 경험, 산업 경험, 평점 등)
- CertBetter의 "Compare ISO Consultants" 기능 참고

#### 2.4 컨설턴트 등록 유도 강화
**제안 사항**:
- 메인 페이지에 "컨설턴트 등록" CTA 버튼 추가 (CertBetter의 "List Your ISO Business" 참고)
- 등록 후 혜택 안내 (예: "검증된 전문가로 등록되면 더 많은 프로젝트 기회를 받을 수 있습니다")

## 💡 핵심 개선 포인트 요약

### 가장 시급한 개선 사항 (Top 3)

1. **필터링 UI 구현** ⚠️
   - API는 있지만 사용자가 활용할 수 없음
   - CertBetter 수준의 필터링 UI 필요

2. **검증 배지 및 관리자 승인 기능** ⚠️
   - 검증 시스템의 핵심 기능
   - 신뢰도 확보를 위해 필수

3. **컨설턴트 프로필 상세 페이지** ⚠️
   - 사용자가 컨설턴트 정보를 자세히 볼 수 있어야 함
   - 매칭 결정에 중요한 정보

## 📝 결론

CertBetter와 비교했을 때, InsightMatch는 **기술적 기반(데이터 모델, API)은 잘 갖춰져 있지만, 사용자 경험(UI/UX) 측면에서 큰 차이**가 있습니다. 특히:

1. **필터링 기능이 API에만 존재**하여 사용자가 활용할 수 없음
2. **검증 시스템이 데이터베이스에는 있지만 UI에 반영되지 않음**
3. **관리자 승인 워크플로우가 없어** 검증 프로세스가 작동하지 않음

**즉시 구현해야 할 핵심 기능**은 필터링 UI, 검증 배지, 관리자 승인 기능입니다. 이를 통해 CertBetter 수준의 사용자 경험을 제공할 수 있을 것입니다.
